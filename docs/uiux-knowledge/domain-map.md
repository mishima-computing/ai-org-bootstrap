# UI/UX Knowledge Domain Map

Status: Cycle 1 foundation artifact. This map is repo-local planning surface and does not add reusable pack knowledge until the manifest delta in [knowledge-architecture.md](knowledge-architecture.md#manifest-delta) lands.

Related docs: [knowledge architecture](knowledge-architecture.md), [absorption ledger](absorption-ledger.md), [authoring program](authoring-program.md).

## Status Rules

| Status | Meaning | Graduation requirement |
| --- | --- | --- |
| Committed | Cycle program will author anchors/cards for this area. | Entry names the deliverable decision it changes and a detection instrument. |
| Stub | Ontology area is present, but authoring is deferred. | Promote only when a concrete deliverable decision, detection instrument, and verified canonical anchors are named. |

Coverage terms: `empty` means no existing `.agent-org/knowledge/ui/` card carries the area. `partial-via-card` means current cards contain narrow incident rules only and must not be treated as complete coverage.

Canonical anchor bodies must resolve to public canonical pointers in later anchor indexes. Incident-memory-only material can be retained as hypothesis evidence, but never as canonical sourcing.
