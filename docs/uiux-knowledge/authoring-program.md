## Acceptance Standard

| Requirement | Standard |
| --- | --- |
| Genius pointer verification | Each cycle verifies public canonical anchors by pointer, version/date, and scope. Fast-moving HIG/M3/convention sources must be dated. |
| Decidable card checks | Each authored or revised card has checks that can be applied from objective intake, screenshot review, source pointer verification, or accessibility testing. |
| No #44 dependency | Exemplar integration is allowed as dated hypothesis evidence, but acceptance cannot depend on perceptual-gate or capture tooling. |
| Prohibition graduation | Every textual prohibition follows [knowledge-architecture.md](knowledge-architecture.md#prohibition-graduation-law). |
| Phrase-check retirement | Existing validator phrase checks retire only when the absorbing area's anchor lands, citations resolve, and [absorption-ledger.md](absorption-ledger.md) records the replacement. |
| No adoption claim | Authoring cycles produce pack artifacts only; downstream adoption is separate. |

## Final Benchmark

The HP one-shot benchmark is the final exam because it should exercise typography, hierarchy, layout, color, accessibility, IA, information design, interaction/motion, composition, genre grammar, process, and evaluation together.

Owner confirmation is required for:

| Question | Needed before final exam |
| --- | --- |
| Scope | Which HP surface, locale, and user task define the benchmark? |
| Rubric | Which observable decisions count as pass/fail evidence? |
| Constraints | Which profiles and anchors are allowed in the one-shot run? |

Until those answers are recorded, the HP benchmark remains the planned final exam, not an assumed acceptance gate.
